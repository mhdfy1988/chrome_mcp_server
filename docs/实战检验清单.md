# 实战检验 TODO 清单

这份清单用来跟踪 `chrome_mcp_server` 的真实网页自动化检验进度。

目标不是只验证“某个工具能调用”，而是验证整条链路是否稳定：

- 页面理解是否准确
- 目标定位是否跑偏
- 动作执行后是否真的发生了预期变化
- 结果验证是否只看 `URL`，还是能看到内容就绪
- 出现新页、弹窗、局部刷新时，返回页是否正确

## 通过标准

每个任务默认至少满足这几条，才算真正通过：

1. 有明确的入口、动作和成功信号
2. 至少保留一张关键步骤截图
3. 失败时能回溯到是“页面理解 / 目标定位 / 动作执行 / 结果验证”哪一层出了问题
4. 搜索或跳转类任务，不能只看 `URL` 变化，必须看到内容区或详情页真的出来
5. 视频类任务，不能只点开详情页，必须验证进入播放态

3. 如有必要，把稳定经验沉淀回 `tests/web-tasks/*.json` 或 `README.md`
